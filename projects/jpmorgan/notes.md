# JPMorgan Chase — Frontend Engineering & UI/UX Leadership Summary

This document summarizes my multi‑year tenure at JPMorgan Chase, where I progressed from an application developer to a lead UI/UX engineer responsible for modernizing internal platforms, shaping user experience strategy, mentoring teams, and delivering mission‑critical tools used across the firm.

My work spanned four major areas:

* Cassandra as a Service (Lead UI/UX Developer)
* Account Lifecycle Management (Lead UI/UX Developer)
* GROOT Technology Graphing Tools (Application Developer)
* Move Money Operations & Visualization Tools (Application Developer)

Across these roles, I consistently designed, architected, and built high‑impact internal products that supported complex financial workflows and helped modernize JPMorgan’s internal engineering ecosystem.

---

## 1. Lead UI/UX Developer — Cassandra as a Service (2019–2021)

I served as the lead frontend and UX developer for JPMorgan’s “Cassandra as a Service” initiative—an internal platform enabling teams across the company to provision, manage, and interact with their distributed Cassandra clusters.

### **Core Responsibilities & Achievements**

* **Designed and built the entire self‑service UI** for internal Cassandra cluster management.
* **Worked directly with end‑users** (engineers, SREs, DBAs) to determine what tools they actually needed and designed the UI around real workflows, not assumptions.
* **Rewrote the legacy UI** from an embedded Spring Boot Angular 1.x interface into a **standalone React + Redux application**, dramatically improving:

  * Load time and responsiveness
  * Code maintainability
  * Scalability as the service expanded
  * Developer onboarding speed
* Established frontend architectural standards and implemented a predictable, modular code structure.
* Delivered a **web‑based database management interface** similar to SQL Server Management Studio, allowing users to:

  * Connect directly to Cassandra clusters
  * Execute CQL commands
  * View schema metadata
  * Inspect keyspaces, tables, and logs

### **Impact**

* Reduced friction for internal teams managing distributed systems.
* Reduced operational overhead by enabling safe self‑service flows.
* Lowered the barrier to entry for new developers by replacing the unmaintainable Angular 1.x codebase.
* Delivered a modern UI that became the standard interface for Cassandra operations inside the firm.

### **Impact**

* Created UI foundations that multiple teams were able to adopt.
* Ensured consistency across applications by defining reusable patterns.

## 3. Application Developer — GROOT (2015–2018)

GROOT was JPMorgan’s internal suite of tools for visualizing the firm’s technology landscape using graph‑based representations.

### **Technical Responsibilities**

* Worked on both greenfield and existing web applications.
* Built interactive graph visualization tools using:

  * AngularJS
  * SCSS
  * Jade templates
  * Gulp build system
* Converted a tightly integrated Grails UI into a **standalone frontend** communicating with services via AJAX.
* Implemented visual tooling to display dependencies, system lineage, and architecture flows.

### **Impact**

* Improved clarity and maintainability by decoupling UI from backend frameworks.
* Enabled better understanding of complex technology relationships inside the bank.

The Move Money team handled batch processing and systems that tracked and processed high‑value transactions across the bank’s infrastructure.

### **Operations & Engineering Responsibilities**

* Diagnosed issues across both AIX and Windows Server environments.

### **Frontend Engineering Contributions**

* Designed and built the **Move Money Dashboard**, including:

  * AngularJS frontend
  * WebSockets live‑updates

### **Impact**
